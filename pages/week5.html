JavaScript Execution Context

Understanding the JavaScript engine, execution context, and call stack is crucial for effective JavaScript development. Each concept plays a significant role in how JavaScript code is interpreted and executed in the browser.

JavaScript Engine

  • Single threaded - only one task at a time
  • Every browser has its own JavaScript Engine
  • They need an engine to understand and execute JavaScript code

Execution Context

  • The Execution Context handles the execution of the code
  • During runtime, the parser parses the source code, allocating memory for variables and functions
  • The source code is then executed

Call Stack

  • The Call Stack uses the LIFO principle to manage function invocation
  • Function calls are placed on the stack; when completed, they are popped off

Databases with MongoDB

Understanding MongoDB's fundamental concepts, including databases, collections, documents, and fields, is essential for working with NoSQL databases.

MongoDB Terms/Concepts

  • Database - A collection of data
  • Collection - A group of MongoDB documents
  • Document - A record in a MongoDB collection
  • Field - A data point within a document
